# Plan: Remove `flat` layout support from `src/renpy-player`

## Overview

Remove the `flat` sprite layout option entirely, leaving `outfit_pose` as the only layout. This simplifies the config surface, removes dead branches in the parser, and eliminates the per-character `layout` field.

## Backward compatibility

- Persisted settings with `"defaultSpriteLayout": "flat"` or `"layout": "flat"` will self-heal: the Zod schema repair in `loadPersistedSettings()` drops unknown/invalid keys and falls back to defaults.
- Characters that were using `flat` layout will now resolve through `outfit_pose` paths (`<assetRoot>/<character>/<outfit>/<pose>/<expression>.png`). This is the intended behavior — users should restructure their asset folders accordingly.
