# Plan: World Wrapper for Camera (Option B) — Fix Sprite Wobble

## Goal

Move camera transform (scale + pan) from individual sprites/background onto a single `.renpy-player__world` wrapper element. This eliminates sprite `left` jitter during camera transitions by keeping sprite layout positions constant and animating only the world's `transform`.

## Files to modify

1. `src/renpy-player/SceneLayer.vue` — add world wrapper + `worldRef`
2. `src/renpy-player/renpy-player.scss` — add `.renpy-player__world` styles
3. `src/renpy-player/useRenpyPlayerController.ts` — `worldStyle`, fix `spriteStyle`/`backgroundStyle`/`stageStyle`/`renderedSprites`
4. `src/renpy-player/player-composables.ts` — `worldElement` ref + `setWorldElement`, update transition tracking

## Why this fixes the wobble

| Before | After |
|--------|-------|
| Camera pan in `--sprite-y` CSS var on each sprite | Camera pan in world wrapper `transform` |
| `left` on sprite shells changes with `backgroundScale` via `zoom` param | `left` is constant (zoom = 1), world handles spatial expansion |
| Multiple CSS transitions fire simultaneously during camera change | Single `transform` on world wrapper, GPU-composited |

## Edge case: `spriteBaselineOffsetPx ≠ 0`

Default is **0**, so `--sprite-y` = 0px and world scaling has no effect on it. If a user sets a non-zero value, the offset will be visually scaled by `backgroundScale` (because it's inside the scaled world). To preserve pixel-exact offsets, add compensation in `stageStyle`:

```ts
'--sprite-y': `calc(var(--renpy-sprite-offset-y, 0px) / ${activeCameraPreset.value.backgroundScale})`,
```

This is optional and can be deferred — default is 0 and most users won't change it.
