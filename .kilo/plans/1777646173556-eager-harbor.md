# Typewriter Effect for Ren'Py Player Dialogue

## Goal

Make the dialogue text in the VN player appear one character at a time, like a typewriter. Clicking during animation reveals the full text immediately; clicking again advances to the next frame.

## Files to Modify

| File | Change |
|------|--------|
| `settings.ts` | Add `typewriterSpeedMs` setting (ms per character) |
| `SettingsPanel.vue` | Add UI control for the new setting |
| `player-composables.ts` | Add `useTypewriter` composable |
| `App.vue` | Wire typewriter into dialogue rendering, stage click, and autoplay |

## Detailed Steps

### 1. Add setting — `settings.ts`

Add to `BaseSettingsSchema`:

```ts
typewriterSpeedMs: z.coerce.number().int().min(0).max(200).default(30),
```

- `0` = instant (disabled), `30` = ~33 chars/sec (default).
- Range 0–200 keeps it reasonable (5 chars/sec minimum).

### 2. Add UI — `SettingsPanel.vue`

Add a "Typewriter speed (ms)" input in the **Animations** section (after "Visibility effect"), following the existing grid pattern:

```html
<div class="renpy-player-settings__field">
  <label>Typewriter speed (ms/char)</label>
  <input v-model.number="settings.typewriterSpeedMs" class="text_pole" type="number" min="0" max="200" step="5" />
  <small>Delay between each character. Set to <code>0</code> to disable.</small>
</div>
```

### 3. Add composable — `player-composables.ts`

Add `useTypewriter`:

```ts
export function useTypewriter(
  sourceText: Ref<string | undefined>,
  speedMs: Ref<number>,
  prefersReducedMotion: Ref<boolean>,
) {
  const displayedText = ref('');
  const isTyping = ref(false);
  const onFinishedCallbacks = new Set<() => void>();
  let charIndex = 0;
  let handle: number | null = null;

  function skip() { ... }
  function reset() { ... }
  function cleanup() { ... }
  function onFinished(cb: () => void) { onFinishedCallbacks.add(cb); }

  watch(sourceText, reset, { immediate: true });
  watch(speedMs, () => { if (isTyping.value) { skip(); reset(); } });

  return { displayedText, isTyping, skip, cleanup, onFinished };
}
```

Key behavior:
- `reset()`: cancels any running interval, reads full text, and if `speedMs > 0` and not reduced-motion, starts a `setInterval` that appends one character per tick. When the last character is reached, calls `onFinishedCallbacks` and sets `isTyping = false`. If `speedMs === 0` or reduced motion, sets `displayedText` to the full text immediately (no callbacks fired since there's no animation).
- `skip()`: cancels the interval and sets `displayedText` to the full source text, `isTyping = false`. Does NOT fire `onFinished` callbacks (skip is a user interrupt, not a natural completion).
- `cleanup()`: cancels interval and clears callbacks (call from `onBeforeUnmount`).
- `onFinished(cb)`: register a callback that fires when the typewriter animation completes naturally.

Also modify `useAutoplay` to accept `isTyping: Ref<boolean>` and change from `setInterval` to `setTimeout`-based scheduling. The `stepForward` callback passed to `useAutoplay` will be wrapped to coordinate with typewriter completion (see step 4d).

### 4. Wire into App.vue

**a) Setup typewriter:**

```ts
const { displayedText: typewriterText, isTyping, skip: skipTypewriter, cleanup: cleanupTypewriter } = useTypewriter(
  computed(() => currentFrame.value?.text),
  computed(() => settings.value.typewriterSpeedMs),
  prefersReducedMotion,
);
```

**b) Template — replace direct text binding:**

```html
<!-- Before -->
<div class="renpy-player__text">{{ currentFrame.text ?? 'No dialogue on this frame.' }}</div>

<!-- After -->
<div class="renpy-player__text">{{ currentFrame ? (typewriterText || '') : 'No dialogue on this frame.' }}</div>
```

When `currentFrame` is null (empty state), show the placeholder as before. When a frame exists, show the typewriter-progressed text.

**c) Stage click — skip typewriter first:**

```ts
function onStageClick() {
  if (!hasFrames.value || isSceneTransitioning.value) return;
  if (isAutoplaying.value) { stopAutoplay(); return; }
  if (isTyping.value) { skipTypewriter(); return; }  // <-- new
  stepForward();
}
```

**d) Autoplay — restart delay after typewriter finishes:**

Instead of the current `setInterval`-based autoplay, switch to a `setTimeout`-based approach that schedules the next advance only after both: (1) typewriter finishes, and (2) the autoplay delay elapses.

Add a `onTypewriterFinished` callback to `useTypewriter` that fires when the typewriter animation completes (not when skipped). In `App.vue`, provide a callback that starts the autoplay delay timer.

In `useAutoplay`, replace `setInterval` with `setTimeout`:

```ts
function scheduleNext() {
  if (!isAutoplaying.value) return;
  if (frameIndex.value >= frames.value.length - 1) { stopAutoplay(); return; }
  autoplayHandle.value = window.setTimeout(() => {
    if (isTyping.value) return; // safety guard, shouldn't normally happen
    stepForward();
    // After stepForward, frame changes → typewriter starts → onTypewriterFinished → scheduleNext
  }, autoPlayDelayMs.value);
}
```

Wire `onTypewriterFinished` to call `scheduleNext()` when both `isAutoplaying` and `!isTyping`:
- When autoplay is active and typewriter finishes → `scheduleNext()` starts the delay timer
- When autoplay is active and typewriter is skipped (user click) → the click stops autoplay (existing behavior), so no conflict
- When autoplay starts (toggle on) → if currently typing, wait for `onTypewriterFinished`; if not typing, `scheduleNext()` immediately

This ensures the delay timer only starts counting *after* the typewriter animation completes.

**e) Cleanup:**

In `onBeforeUnmount`, add `cleanupTypewriter()`.

## UX Behavior Summary

| Action | Result |
|--------|--------|
| Frame changes with `typewriterSpeedMs > 0` | Text reveals one char at a time |
| Frame changes with `typewriterSpeedMs = 0` | Text appears instantly (as before) |
| Click/tap during typewriter animation | Full text revealed immediately |
| Click/tap after typewriter finishes | Advances to next frame |
| Autoplay + typewriter finishes | Autoplay delay timer starts *after* text fully revealed |
| Autoplay toggle while typing | Waits for typewriter to finish, then starts delay |
| `prefers-reduced-motion` active | Typewriter disabled, instant text |
