## Changes

### 1. Add new section: `## Runtime Environment` (insert after `## Purpose`)

Explain:

- **Module type**: renpy-player is a Tavern Helper **script** (only `index.ts`, no `index.html`). It does not render inside an iframe UI. Instead it mounts Vue components directly onto the SillyTavern host page using jQuery.
- **iframe consequences**: Styles compiled inside the iframe don't reach the host page. `teleportStyle()` is called once in `index.ts` to copy all `<style>` elements into the host `<head>`. This is why the module uses unscoped SCSS with BEM-prefixed `renpy-player__` classes — NOT tailwindcss (tailwind class names would collide with SillyTavern's existing CSS).
- **jQuery scope**: In scripts, `$` operates on the host SillyTavern page (`window.$ = window.parent.$`), so `$('#chat')` selects the real chat element.
- **Lifecycle**: Init inside `$(() => { ... })` (never `DOMContentLoaded` — it doesn't fire for dynamically-loaded scripts). Cleanup via `$(window).on('pagehide', ...)`. The controller also uses Vue `onMounted`/`onScopeDispose` because it's created inside `App.vue`'s `<script setup>`.

### 2. Add new section: `## Build System & Auto-Imports` (insert after `## Runtime Environment`)

Explain:

- **Auto-imported (no import statement needed)**:
  - Vue: `ref`, `computed`, `watch`, `watchEffect`, `onMounted`, `onScopeDispose`, `onBeforeUnmount`, `createApp`, `createPinia`, `defineStore`, `storeToRefs`, `inject`, `provide`, `reactive`, `readonly`, and all other Vue Composition API exports.
  - Zod: `z` (the root zod namespace).
  - `klona` (deep-clone utility for stripping Vue Proxy before persistence).
  - VueUse functions via `@vueuse/core` auto-import.
  - VueUse components/directives via `unplugin-vue-components`.
- **NOT auto-imported** (require explicit import):
  - Local modules (`./parser`, `./types`, etc.)
  - Utility functions from `@util/script` (e.g., `createScriptIdDiv`, `teleportStyle`)
  - Vue components from other `.vue` files
- **Options API is disabled** (`__VUE_OPTIONS_API__ = false`). Always use Composition API with `<script setup lang="ts">`.
- **Path alias**: `@util/` resolves to `util/` at project root (e.g., `import { createScriptIdDiv } from '@util/script'`).

### 4. Add new section: `## DOM Mounting` (insert after `## Tavern Helper Globals`, before existing `## Integration`)

Explain the mounting mechanism in detail:

- **`createScriptIdDiv()`** (from `@util/script`): creates `<div script_id="...">`. Tavern Helper uses `script_id` for scoped DOM management and auto-cleanup.
- **Player host**: Created with `attr('id', 'th-renpy-player')` and inserted before `$('#chat')`. Must be re-attached on `CHAT_CHANGED` and `MORE_MESSAGES_LOADED` because SillyTavern rebuilds the chat DOM on those events.
- **Settings host**: Appended to `#extensions_settings2` (SillyTavern's standard extension settings area).
- **Shared Pinia**: A single `createPinia()` instance is created in `index.ts` and `.use()`'d on both `playerApp` and `settingsApp`, so they share the same store state.
- **`teleportStyle()`**: Clones all `<style>` from the iframe `<head>` into a `<div script_id="...">` appended to the host `<head>`. `destroy()` removes that div. Called in `index.ts`.

### 7. Add new section: `## SillyTavern CSS Integration` (insert after `## Runtime Behavior`)

Explain:

- The SCSS reads SillyTavern host-page CSS custom properties for theme-aware styling:
  - `--SmartThemeBorderColor` — theme border color
  - `--SmartThemeQuoteColor` — theme quote/accent color
  - `--black30a`, `--white30a` — semi-transparent black/white
- These are NOT defined by the renpy-player. They come from SillyTavern's theme engine and are available on the host page.
- `color-mix(in srgb, ...)` is used to blend these with transparency.
- Internal CSS variables (`--renpy-*`, `--sprite-*`) are set by the controller via inline `stageStyle` computed.
