# Fix: Speaker name flickering on named→named transitions

## Problem

`visibleSpeaker` is a live computed from `currentFrame`, so the DOM text updates *before* the opacity animation finishes. When transitioning `mirai` → `shino`:
1. `currentFrame` changes → DOM immediately shows "shino"
2. `hasSpeakerChange` fires → `speakerRevealed = false` → opacity animates to 0 then back to 1
3. Result: "shino" briefly visible, then fades out, then fades in = flicker

## Desired behavior

| Transition | Effect |
|---|---|
| Same speaker (A → A) | No animation, instant |
| Different named (A → B) | Instant swap, no animation |
| Named → narrator (A → '') | Fade out the name |
| Narrator → named ('' → B) | Fade in the new name |

## Changes (3 files)

### 1. `src/renpy-player/player-composables.ts` — `useDialogueReveal`

- Add `displayedSpeaker` ref (decoupled from `currentFrame`)
- In `startReveal`, classify transitions into three categories:
  - `hasSpeakerGone` (named → empty): keep old name, fade out, clear after timeout
  - `hasSpeakerAppeared` (empty → named): set new name, fade in after timeout
  - Everything else (same or named→named): instant swap, no animation
- Only apply `speakerIntroDelay` when `hasSpeakerAppeared`
- In `clearReveal`, reset `displayedSpeaker` to `''`
- In the `currentFrame` watcher null branch, reset `displayedSpeaker` to `''`
- In the `effectsDisabled` watcher, set `speakerRevealed` based on `displayedSpeaker !== ''`
- Export `displayedSpeaker`

### 2. `src/renpy-player/useRenpyPlayerController.ts`

- Remove the `visibleSpeaker` computed (line 137)
- Add `displayedSpeaker` to the destructure from `useDialogueReveal`
- In the returned `reactive({...})`, replace `visibleSpeaker` with `displayedSpeaker` in the `dialogue` group

### 3. `src/renpy-player/ViewportOverlay.vue`

- Change template from `controller.dialogue.visibleSpeaker` to `controller.dialogue.displayedSpeaker` (line 16)
