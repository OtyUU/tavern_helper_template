# Asset Resolution Diagnostics â€” Show Loaded/Failed

**Goal:** Replace the noisy raw "Candidates:" list in DiagnosticsPanel with actionable info: which candidate actually loaded, and which ones failed (only shown if failures exist).

## Current behavior

- `SmartImage.vue` iterates candidates internally via `resolveFirstCandidate()`, logging failures to console only.
- `DiagnosticsPanel.vue` dumps the full candidate list (often 8+ paths) with no indication of which resolved.

## Plan

### 1. `SmartImage.vue` â€” Track & emit resolution results

- Add a `failedCandidates: string[]` ref.
- In `resolveFirstCandidate()`, collect failed candidate URLs into the array instead of just `console.warn`.
- After resolution completes (success or exhaustion), emit a new event:

```ts
emit('resolutionStatus', { resolved: string | null, failed: string[] })
```

- Reset `failedCandidates` at the start of each `syncCurrentSrc()` call.
- Emit `resolutionStatus` in `syncCurrentSrc()` after `resolveFirstCandidate` returns (both success and all-exhausted paths).
- Also emit from `handleDisplayError()` since a display-time failure triggers re-resolution with the current src blocked.

### 2. `useRenpyPlayerController.ts` â€” Store resolution status

- Add a reactive map:

```ts
const assetResolutionStatus = ref<Record<string, { resolved?: string; failed: string[] }>>({});
```

- Add handler:

```ts
function onAssetResolutionStatus(key: string, status: { resolved?: string | null; failed: string[] }) {
  assetResolutionStatus.value[key] = { resolved: status.resolved ?? undefined, failed: status.failed };
}
```

- Expose `assetResolutionStatus` and `onAssetResolutionStatus` in the `diagnostics` group.

### 3. `SceneLayer.vue` â€” Wire the new event

- On the **background** `SmartImage`, add:
  ```html
  @resolution-status="controller.diagnostics.onAssetResolutionStatus('__background__', $event)"
  ```
- On the **sprite** `SmartImage`, add:
  ```html
  @resolution-status="controller.diagnostics.onAssetResolutionStatus(sprite.id, $event)"
  ```

### 4. `DiagnosticsPanel.vue` â€” Clean display

Replace the current candidate list sections with:

**Sprites** â€” for each sprite:
- Show the resolved src (if any): `âś“ path/to/resolved.png`
- Show failed candidates (if any): `âś— path/to/failed.png`
- Show "No asset candidates" if the candidate list was empty
- Show "Not resolved" if candidates exist but none loaded

**Background** â€” same pattern using `__background__` key.

Only failed candidates are listed â€” successful resolution of the first candidate shows just the one `âś“` line.
