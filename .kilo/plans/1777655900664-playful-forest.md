# Plan: Speaker UI Behavior for Ren'Py Player

## Problem

The current speaker UI has two issues:

1. **Narrator frames show "Narrator"** — when bare quoted text (`"Hello"`) is parsed, the speaker is set to `'narrator'`, which `getSpeakerLabel()` capitalizes to `"Narrator"` and renders visibly in the speaker column.
2. **Speaker column disappears when empty** — the CSS uses `--no-speaker` modifier class that changes `grid-template-columns` from two columns to one, causing the text column to shift position.

## Desired Behavior

- Frame with `speaker: "chinami"` → speaker column shows "chinami", text column shows dialogue
- Frame with `speaker: null` or `speaker: ""` → speaker column is empty (no text, no element), text column stays in same position
- Text column width and position is identical whether speaker is present or not
- The word "Narrator" never appears anywhere in the player UI

## Changes

### 1. `src/renpy-player/parser.ts` — Map narrator to empty speaker

**Line 78** — In `getSpeakerLabel()`, return `''` instead of `'Narrator'`:

```ts
// Before:
if (raw.toLowerCase() === 'narrator') return 'Narrator';

// After:
if (raw.toLowerCase() === 'narrator') return '';
```

This handles the `dialogue` command path (line 685) where `getSpeakerLabel(cmd.speaker)` is called. Narration lines (bare `"text"`) set `speaker: 'narrator'` at line 302, so `getSpeakerLabel('narrator')` now returns `''`.

### 2. `src/renpy-player/App.vue` — Remove the `--no-speaker` modifier and keep two-column grid always

**Template (lines 59–63)** — Remove the conditional class and always keep the speaker div rendered (but empty when no speaker):

```vue
<!-- Before: -->
<div class="renpy-player__dialogue-bar"
  :class="{ 'renpy-player__dialogue-bar--no-speaker': !visibleSpeaker }">
  <div v-if="visibleSpeaker" class="renpy-player__speaker">{{ visibleSpeaker }}</div>
  <div class="renpy-player__text">...</div>
</div>

<!-- After: -->
<div class="renpy-player__dialogue-bar">
  <div class="renpy-player__speaker">{{ visibleSpeaker }}</div>
  <div class="renpy-player__text">...</div>
</div>
```

**Style (lines 937–940)** — Delete the `.renpy-player__dialogue-bar--no-speaker` rule entirely, since the grid is always two columns now:

```scss
// DELETE this block:
.renpy-player__dialogue-bar--no-speaker {
  grid-template-columns: 1fr;
  gap: 0;
}
```

This ensures the text column always occupies the same grid cell regardless of whether a speaker is present.

### 3. `src/renpy-player/App.vue` — Ensure `visibleSpeaker` returns empty string (not null) when absent

**Lines 317–320** — Change the computed to return `''` instead of `null` when there's no speaker, so the template renders an empty div rather than needing a conditional:

```ts
// Before:
const visibleSpeaker = computed(() => {
  const speaker = currentFrame.value?.speaker?.trim();
  return speaker || null;
});

// After:
const visibleSpeaker = computed(() => {
  return currentFrame.value?.speaker?.trim() ?? '';
});
```

### 4. `src/renpy-player/parser.ts` — Ensure preview frames also never show "Narrator"

The preview frames at lines 708 and 718 use hard-coded `'Scene Preview'` / `'Active Scene'` speaker values, which are fine — they are not "Narrator". No change needed here.
