# Fix: Camera sprite scale should match background scale (unified zoom)

## Problem

In `index.html`, a single `cameraZoom` scales both background and sprites together. The RenPy player uses separate `backgroundScale` and `spriteScale` values, causing sprites to render at a different scale than the background (e.g., closeup: bg=1.5, sprite=1.8 → sprite is 20% larger than background).

## Changes

### 1. `src/renpy-player/useRenpyPlayerController.ts`

**1a. `spriteStyle` computed (line 728–733)**
- Change `camera.spriteScale` → `camera.backgroundScale` for `--sprite-scale`
- This makes the sprite zoom match the background zoom (unified camera zoom)

**1b. `renderedSprites` computed (line 777–795)**
- Change `const zoom = activeCameraPreset.value.spriteScale` → `activeCameraPreset.value.backgroundScale`
- The `zoom` variable is passed to `getSpriteShellStyle(position, zoom)` which calculates X anchor position. Using `backgroundScale` ensures position scaling matches the unified camera zoom.

### 2. `src/renpy-player/SettingsPanel.vue`

**2a. Default card (lines 190–197)**
- Remove the "Sprite scale" input (`settings.defaultSpriteScale`)
- Rename "Background scale" label → "Camera scale"

**2b. Medium card (lines 228–239)**
- Remove the "Sprite scale" input (`settings.mediumSpriteScale`)
- Rename "Background scale" label → "Camera scale"

**2c. Closeup card (lines 243–256)**
- Remove the "Sprite scale" input (`settings.closeupSpriteScale`)
- Rename "Background scale" label → "Camera scale"

### 3. `src/renpy-player/settings.ts` — no schema changes

The `*SpriteScale` fields (`defaultSpriteScale`, `mediumSpriteScale`, `closeupSpriteScale`) remain in the Zod schema for backward compatibility. They are simply no longer read by the controller or exposed in the UI. Existing saved values in tavern variables won't break; they just become inert.
