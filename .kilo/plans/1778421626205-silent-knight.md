# Plan: SmartImage downscale + mip cache + targetHeightPx

## Overview

Two changes: (1) upgrade `SmartImage.vue` with canvas-based downscale and per-component mip cache, (2) wire `targetHeightPx` from controller through to the component for sprites.

---

## Step 1 — SmartImage.vue

**File:** `src/renpy-player/SmartImage.vue`

### 1a. Template change
Add `ref="rootRef"` to the root `<div>`:
```html
<div ref="rootRef" class="smart-image" :style="smartImageStyle">
```

### 1b. Script replacement
Replace the entire `<script setup lang="ts">` block (lines 22–168) with the new version provided in the spec. Key differences:

| Aspect | Before | After |
|---|---|---|
| Props | `candidates`, `alt`, `swapDurationMs` | + `downscale`, `targetHeightPx`, `downscaleOversize`, `downscaleMinRatio` |
| Refs | `currentSrc`, `previousSrc`, etc. | + `rootRef`, `resolvedCandidateSrc` |
| Watch trigger | `() => [...props.candidates]` | `[candidateKey, () => props.downscale, () => props.targetHeightPx]` |
| Image preload | Returns `{ src, naturalWidth, naturalHeight }` | Returns `CandidateMeta` (same fields + `image: HTMLImageElement`) |
| Display flow | Shows resolved candidate directly | Shows original immediately, then async-swaps to downscaled mip blob |
| Mip cache | None | `Map<string, MipEntry[]>` per component, max 4 entries/src |
| Cleanup | Only clears swap timeout | Also revokes all cached blob URLs |

No `<style>` changes.

